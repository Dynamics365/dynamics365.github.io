Thông thường khi ta chạy một form trong Dynamics AX 2012 lần đầu tiên, ta sẽ thấy thời gian nạp khá lâu do các thủ tục biên dịch và đưa vào phần đệm của AOS.

và nếu chúng ta có nhu cầu khởi động lại AOS thường xuyên ví dụ như mỗi cuối ngày, hay một môi trường demo chỉ start lên khi có nhu cầu, thì chúng ta phải làm thế nào để những form thường dùng vẫn đảm bảo tốc độ.

Giải pháp ở đây là sau khi start AOS chúng ta sẽ cho hệ thống chạy một đoạn script mở những trang thường dùng lên, cache vào AOS, sau đó đóng chúng đi, làm như vậy thì người dùng sẽ cảm thấy hệ thống trơn tru hơn :)

static void WarmupRF(Args _args)

{
    UtilElements e;
    TreeNode treeNode;
    FormRun formRun;

    Args args = new Args();

    while select e
        where e.utilLevel  == UtilEntryLevel::var    //<-- specify layer here
        &&    e.recordType == UtilElementType::Form  //<-- and only forms
        &&    e.name like "nameofformPrefix"

        {
            try
            {
                treeNode = xUtilElements::getNodeInTree(xUtilElements::parentElement(e));
                args.name(treeNode.AOTname());
                formRun = ClassFactory.formRunClass(args);
                formRun.init();
                //formRun.run();   //<-- No need to run the form, but sometimes it can load the data
                formRun.close();
            }
            catch
            {
                Infolog.clear();
                continue;
            }
        }
}
