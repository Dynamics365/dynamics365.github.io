Running
the AIF Document Service Wizard

Three ways to
start AIF Document Service Wizard:

1.   
In the Development Work space, click on Tools > Wizards
> AIF Document Service Wizard

2.   
Tools > Application Integration Framework > Create
document service.

3.   
Right-clicking on a query, then clicking
on Add-Ins | AIF Document Service Wizard

How to do?

[more]

Step 1: Tools > Wizards >
AIF Document Service Wizard, then click Next


Step 2: Fill in information for document, then click Next
Step 3: Select code generation parameters, then click Next
Click on Generate button to finish

The output of
the AIF document Service Wizard is stored in a private project that has the
same name as the query that the service is based on.


In the client
user interface:

  1. System administration > Setup > Services
    and application integration framework > Outbound ports


       2.  Click
New to create a new port, and fill in required information

  • Port
    Name: name of the port
  • Description:
    a description for the port
  •  Adapter:
    select a suitable adapter (Users normally choose File system adapter)
  • URI:
    identify the address/direction for exported file
  • Check
    Expose service operation and Customize documents

  • Click on Service
    operations and choose needed operations (Select operations have the same name
    as Service class, which is generated in the private project)





Click on Data policies and choose fields for exporting



             3. Save and click on Active button




The following
code is used for execute the AIF file outbound Service

          1. Create
a record in AIF Outbound Service

  1. AxdSendContext          axdSendContext      = AxdSendContext::construct();
  2. AifAction               aifAction ;
  3. AifConstraint           aifConstraint       = new AifConstraint();
  4. AifConstraintList       aifConstraintList   = new AifConstraintList();
  5. AifOutboundProcessingService AifOutboundProcessingService =new AifOutboundProcessingService();
  6.  
  7. AifEndpointList endpointList;
  8.  
  9. Query query;
  10. QueryBuildDataSource      queryBuildDataSource;
  11. ;
  12.  
  13. //Select datasource
  14. query = new Query();
  15. queryBuildDataSource = query.dataSourceTable(tablenum(SalesTable));
  16.  
  17. //#Class SalesTableService was created when outbound port created (located in the private project)
  18. aifAction = AifAction::find(AifSendService::getDefaultSendAction(classnum(SalesTableService), AifSendActionType::SendByQuery)) ;
  19.  
  20. //Select context as original
  21. axdSendContext.parmXMLDocPurpose(XMLDocPurpose::Original);
  22. axdSendContext.parmSecurity(false);
  23.  
  24. //No constraint
  25. aifConstraint.parmType(AifConstraintType::NoConstraint);
  26. aifConstraintList.addConstraint(aifConstraint) ;
  27.  
  28. endPointList=AifSendService::getEligibleEndpoints(aifAction.ActionId,aifConstraintList);
  29.  
  30. AifSendService::submitFromQuery(aifAction.ActionId, endpointList, query, AifSendMode::Sync);


The result of this code is displayed in Queue manager form: System administration > Periodic >
Services and Application Integrate Framework > Queue manager
